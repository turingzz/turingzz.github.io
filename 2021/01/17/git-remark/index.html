git常用命令

Author: turingzz Date: January 17, 2021  21:08:09 Category: study

版本管理

  1. 新建一个空目录，使用git init将该目录变为可以git可以管理的目录
  2. 创建了一个新文件readme.md，使用git add readme.md添加到仓库
  3. 可以在多次add之后，一次性提交，使用git commit -m "add a readme file"
  4. 使用git status可以查看当前的目录的状态，有修改，有可以提交的等等
  5. 当文件add后，又经过修改切暂未add，使用git diff filenae可以看到这个文件修改的地方
  6. git log查看提交记录。git log --graph简易的图示提交记录
  7. git reset --hard HEAD^回退到上一个版本，那么想取消回退呢。要先找到刚刚的版本号，然后git reset --hard fc58。**使用git reflog**可以查看每一次记录，包括“未来”的记录
  8. git reset HEAD <file>可以从暂存区撤销回工作区。git checkout -- <file>可以找回删除的某个文件（用版本库里的版本替换工作区的版本），无论工作区是修改还是删除，都可以“一键还原”。

远程仓库

  1. 添加远程仓库：git remote add origin git@github.com:username/learngit.git(origin可以替换为不同的名字如gitee，github等等)
  2. 将本地的master推送到远程库origin：git push -u origin master（第一次推送加-u，之后不需要)
  3. git fetch origin获取它有，但是本机没有的代码。fetch之后要merge到本地分支，git merge origin/master
  4. 查看当前的远程仓库git remote -v
  5. 删除仓库git remote rm origin

分支管理

  1. 创建新的分支并切换到该分支git checkout -b dev。相当于两条命令git branch dev和git checkout dev。切换也可以用git switch -c dev和git switch dev
  2. git branch查看当前的分支
  3. 在master上合并dev。先切换会master，然后git merge dev
  4. 删除dev分支git branch -d dev
